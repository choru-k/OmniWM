# F15 chord layer

Native support for an **F15-held chord layer** + **double-tap-F15 leader**, replacing the external
Hammerspoon setup.

F15 is not a macOS modifier, so Carbon's `RegisterEventHotKey` can't express "hold F15 + key". This
uses a **standalone CGEvent tap** (the `RCmd→F15` mapping comes from Karabiner). It runs alongside
OmniWM's own hyper-trigger tap rather than being woven into it.

## Behavior
- **Hold F15 + key** → run an OmniWM command.
- **Double-tap F15** → open the [Leader](leader.md) tab of the Command Palette.

## Default chord map (v1, hardcoded)
| Key | Action |
|-----|--------|
| `h` `l` `j` `k` | focus left / right / down / up |
| `⇧h` `⇧l` `⇧j` `⇧k` | move window left / right / down / up |
| `f` | toggle column full width |
| `c` | expand column to available width |
| `0` | cycle column width |
| `-` | consume window into column |
| `=` | expel window from column |
| `esc` | cancel the layer |

> The chord list is hardcoded for now (TOML-configurable chords deferred — see the `// ponytail:`
> note in `F15ChordEngine.swift`). The leader (double-tap) menu *is* fully configurable.

## Config (`~/.config/omniwm/settings.toml`)
```toml
[general]
f15Enabled = true          # default false
f15DoubleTapSeconds = 0.3  # double-tap window
```

## Permissions
The tap needs **Input Monitoring** (Privacy & Security → Input Monitoring), separate from
Accessibility. Without the grant the tap simply isn't installed — F15 does nothing, everything else
keeps working (no crash).

## Implementation
- `Sources/OmniWM/Core/Input/F15ChordEngine.swift` — pure state machine: hold tracking (3s stale-timeout), chord lookup, double-tap detection, escape cancel. Unit-tested (`Tests/OmniWMFeatureTests/F15ChordEngineTests.swift`).
- `Sources/OmniWM/Core/Input/F15EventTap.swift` — `@MainActor` CGEvent-tap owner; re-enables on `tapDisabledByTimeout`, dispatches through `WMController.commandHandler`.
- Wired in `WMController.applyPersistedSettings` (configured + installed when `f15Enabled`). Double-tap dispatches `HotkeyCommand.openLeader`.
